# auto_ssh_auther 현재 구현 반영 계획서

## 1. 프로젝트 목적
- 이 앱은 로컬 `~/.ssh` 안의 공개키를 선택해서, 사용자가 직접 입력한 원격 서버의 `authorized_keys`에 등록해주는 GUI 도구다.
- 원격 등록 성공 또는 이미 등록된 키일 때 로컬 `~/.ssh/config`도 자동 반영해, 이후 `ssh <Host>` 접속에서 선택한 키가 자동 사용되도록 한다.
- 첫 목표는 복잡한 운영 관리가 아니라, SSH 공개키 등록과 접속 준비 작업을 간단하고 빠르게 처리하는 것이다.
- 데이터베이스나 JSON 기반 이력 저장은 넣지 않고, 세션 단위로 동작하는 가벼운 앱으로 유지한다.

## 3. 현재 코드 구조
- `main.py`

- `build.py`

- `auto_ssh_auther.spec`

- `src/ssh_auther/ui/main_window.py`
  - 메인 윈도우
  - 키 생성 다이얼로그
  - 연결 테스트 / 키 등록 / 키 생성 / 키 삭제 버튼 처리

  - 공개키 탐색

  - 키 파일 삭제
- `src/ssh_auther/ssh/remote.py`
  - `paramiko` 기반 SSH 연결
  - 원격 명령 실행
  - `authorized_keys` 읽기 / 추가
- `src/ssh_auther/ssh/local_config.py`
  - 로컬 OpenSSH config Host 블록 탐색
  - Host별 접속 옵션 추가/갱신
  - 선택된 공개키에 대응하는 비밀키 경로 계산
- `src/ssh_auther/services/register.py`
  - 키 중복 검사
  - 등록 흐름 제어
  - 로컬 SSH config 반영 연결
  - 접속 테스트 결과 메시지 정리

## 5. 리뷰 결과
### 좋은 점
- 현재 범위가 단순해서 목적과 구현이 잘 맞는다.
- 기능 분리는 `ui`, `keys`, `ssh`, `services` 정도로 깔끔하다.
- 원격 등록 후 로컬 SSH config까지 이어져 실제 접속 자동화 흐름이 완성됐다.
- 키 등록 전에 연결 테스트를 따로 둘 수 있어서 사용자 입장에서 이해하기 쉽다.
- 키 생성/삭제까지 포함되어 실제 사용성은 계획보다 좋아졌다.

### 보완이 필요한 부분
- 타임아웃/연결 실패 예외 처리가 충분히 정확하지 않다.
  - 주요 케이스는 개선했지만, 실제 운영 환경에서 더 다양한 네트워크 예외 메시지를 다듬을 여지는 남아 있다.
- 자동 테스트 범위가 아직 좁다.
  - 현재는 문자열 처리, 오류 메시지, 로컬 config 파일 갱신 위주의 테스트다.
- 원격 추가 로직은 더 명확하게 정리할 수 있다.
  - 개행 보정은 들어갔지만, 장기적으로는 원격 파일 처리 로직을 더 구조화할 여지는 있다.
- 로컬 SSH config는 실제 사용자 파일이 다양하므로 복잡한 `Match`/와일드카드 Host 사례를 추가로 확인할 필요가 있다.

## 7. v1 완료 기준
- 사용자가 로컬 공개키를 선택할 수 있어야 한다.
- 사용자가 서버 주소, 포트, 계정, 비밀번호를 입력할 수 있어야 한다.
- 연결 테스트가 동작해야 한다.
- 버튼 한 번으로 원격 `authorized_keys`에 키를 등록할 수 있어야 한다.
- 이미 등록된 키는 중복 추가되지 않아야 한다.
- 등록 성공 후 로컬 `~/.ssh/config`에 선택한 키가 Host별로 자동 반영되어야 한다.
- 키 생성과 키 삭제가 로컬 `~/.ssh` 기준으로 정상 동작해야 한다.
- 실패 메시지가 사용자가 이해 가능한 수준이어야 한다.

- `uv run python -m unittest discover -s tests -v` 통과

- 로컬 SSH config 변경 없음 테스트 통과

- Python 컴파일 스모크 통과 이력 있음
- 오프스크린 GUI 초기화 스모크 통과 이력 있음
- macOS `PyInstaller` 빌드 통과 이력 있음
- `dist`에 아이콘 리소스 자동 포함 확인 이력 있음

### 아직 필요한 확인
- 실제 테스트 서버에 등록 성공 확인
- 등록 후 `ssh <Host>`로 선택한 키가 자동 사용되는지 확인
- 잘못된 비밀번호 입력 시 메시지 확인
- 존재하지 않는 서버/포트 실패 처리 확인
- 중복 키 등록 시 안내 메시지 확인
- 생성한 키가 즉시 목록에 반영되는지 확인
- 삭제 시 공개키/비밀키가 함께 제거되는지 확인
